## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

- **Node.js** `>= 18.x`
- **npm** `>= 9.x` or **yarn** / **pnpm**
- A **Supabase** account → [supabase.com](https://supabase.com)
- A **Razorpay** account (for payment features) → [razorpay.com](https://razorpay.com)

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/LennyDany-03/blinko.git
cd blinko
```

**2. Install dependencies**

```bash
npm install
```

**3. Set up environment variables**

```bash
cp .env.local.example .env.local
```

Then fill in your credentials (see [Environment Variables](#-environment-variables) below).

**4. Set up Supabase** (see [Supabase Setup](#-supabase-setup) below)

**5. Run the development server**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. 🎉

## 🔐 Environment Variables

Create a `.env.local` file in the root of your project. Here's the full template:

```env
# ─────────────────────────────────────────────
# SUPABASE
# ─────────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# ─────────────────────────────────────────────
# AI (OpenAI / Gemini / Your Provider)
# ─────────────────────────────────────────────
AI_API_KEY=your_ai_provider_api_key
AI_MODEL=gpt-4o

# ─────────────────────────────────────────────
# RAZORPAY (Payments)
# ─────────────────────────────────────────────
NEXT_PUBLIC_RAZORPAY_KEY_ID=your_razorpay_key_id
RAZORPAY_KEY_SECRET=your_razorpay_key_secret

# ─────────────────────────────────────────────
# NEXT.JS
# ─────────────────────────────────────────────
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXTAUTH_SECRET=your_nextauth_secret_here

# ─────────────────────────────────────────────
# GOOGLE OAUTH (via Supabase)
# ─────────────────────────────────────────────
# Configure these inside your Supabase Auth settings dashboard.
# No extra env variables needed — Supabase handles the OAuth flow.
```

> ⚠️ **Never commit your `.env.local` file.** It's already in `.gitignore`.

## 🗄️ Supabase Setup

**1. Create a new Supabase project** at [app.supabase.com](https://app.supabase.com)

**2. Run the database migrations**

```bash
# Install the Supabase CLI
npm install -g supabase

# Login
supabase login

# Link your project
supabase link --project-ref your-project-ref

# Push migrations
supabase db push
```

**3. Enable Google OAuth**

In your Supabase dashboard:
- Go to **Authentication → Providers → Google**
- Add your **Google Client ID** and **Client Secret**
- Set the redirect URL to: `https://your-project.supabase.co/auth/v1/callback`

**4. Configure Row-Level Security (RLS)**

All RLS policies are included in the migration files. After running `supabase db push`, your tables will be secured automatically.

**5. Get your API keys**

In **Project Settings → API**, copy:
- `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
- `anon public` key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `service_role` key → `SUPABASE_SERVICE_ROLE_KEY` *(keep this secret)*

## 🤝 Contributing

Contributions make open source amazing. Every bug fix, feature suggestion, and documentation improvement helps Blinko grow. **We welcome contributors of all skill levels.**

### How to Contribute

**1. Fork the repository**

```bash
git clone https://github.com/YOUR_USERNAME/blinko.git
cd blinko
```

**2. Create a feature branch**

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/bug-description
```

**3. Make your changes**

Follow the [code style guide](#code-style) and write clean, documented code.

**4. Commit with a clear message**

```bash
git commit -m "feat: add QR code generator to profile page"
# Commit types: feat | fix | docs | style | refactor | test | chore
```

**5. Push and open a Pull Request**

```bash
git push origin feature/your-feature-name
```

Then open a PR on GitHub with a clear description of what you changed and why.

### Code Style

- **TypeScript** everywhere — no `any` types
- **Prettier** for formatting (config included)
- **ESLint** rules enforced on CI
- Component names in **PascalCase**, files in **kebab-case**
- Keep components small and composable

### Good First Issues

Look for issues tagged [`good first issue`](https://github.com/LennyDany-03/blinko/issues?q=is:issue+label:good+first+issue) — these are hand-picked for new contributors.

### Reporting Bugs

Open an issue with:
- A clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if relevant
- Your environment (OS, Node version, browser)
